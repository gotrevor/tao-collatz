## Gotchas this lap

- `linarith` chokes on `0.09`-style OfScientific literals — rewrite to
  fractions (`show (0.09:ℝ) = 9/100`) before linear arithmetic.
- Big-context `nlinarith` = heartbeat timeouts; use `linarith only [...]` +
  explicit `mul_le_mul` product hints. `set_option maxHeartbeats` goes BEFORE
  the docstring, not between it and the theorem.
- A trailing in-tactic `calc` greedily consumes following dedented `have`
  lines ("invalid calc step") — end sub-proofs with `exact`, not `calc`.
- `∑'` binders: one paren group per binder (`∑' (a : X) (b : X),`;
  `∑' (a b : X),` is a parse error).
- ℝ≥0∞ end-to-end for PMF tail bounds = zero summability side conditions:
  `ENNReal.tsum_prod'`/`tsum_mul_left`/`tsum_eq_iSup_sum` + `ofReal` only at
  the boundary (`ENNReal.toReal_le_of_le_ofReal`). This pattern carried all
  four tail proofs; reuse for X10b.
- `Real.log_le_rpow_div : log x ≤ x^ε/ε` is the tool for
  `log-power ≪ polynomial` thresholds; `Real.log_le_sub_one_of_pos` both ways
  gives `log(16/9) ≥ 7/16`-style numeric log bounds without exp estimates.
